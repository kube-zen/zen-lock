# Maintainers

This file lists the maintainers of the zen-lock project.

## Current Maintainers

### Kube-ZEN Community

The zen-lock project is maintained by the Kube-ZEN community.

## Maintainer Responsibilities

Maintainers are responsible for:

- **Code Review**: Reviewing pull requests for quality, correctness, and adherence to project standards
- **Issue Management**: Triaging issues, labeling appropriately, and ensuring they are addressed
- **Release Management**: Making decisions about releases and maintaining release notes
- **Documentation**: Ensuring documentation is accurate and up-to-date
- **Security**: Responding to security vulnerabilities and maintaining security best practices
- **Community**: Representing the project in the community and helping contributors

## Becoming a Maintainer

Maintainers are added based on:

- Consistent, high-quality contributions to the project
- Demonstrated understanding of the project's goals and architecture
- Willingness to take on maintainer responsibilities
- Consensus among existing maintainers

If you're interested in becoming a maintainer, please reach out to the current maintainers.

## Contact

For questions about maintainership or the project, please open an issue or contact the maintainers.
