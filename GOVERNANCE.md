# Governance

This document describes the governance model for the zen-lock project.

## Project Status

zen-lock is currently in **alpha** status. The project is maintained by the Kube-ZEN community.

## Maintainers

Maintainers are responsible for:
- Reviewing and merging pull requests
- Triaging issues
- Maintaining code quality and project standards
- Making release decisions
- Representing the project in the community

### Current Maintainers

- Kube-ZEN Community

## Decision Making

### Consensus-Based Decisions

Most decisions are made through consensus among maintainers. When consensus cannot be reached, decisions may be escalated to the project's primary maintainers.

### Pull Request Process

1. All pull requests require at least one maintainer approval
2. All pull requests must pass CI checks
3. All pull requests must follow the project's code style and standards
4. Maintainers may request changes before merging

### Issue Triage

- Issues are triaged by maintainers
- Issues should be labeled appropriately (bug, enhancement, question, etc.)
- Issues may be closed if they are duplicates, out of scope, or resolved
